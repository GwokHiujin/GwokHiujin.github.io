『中端优化』Tree-Height Balancing: 树高平衡

最近需要做一些与并行计算相关的代码生成优化工作，因此回顾了一些经典的指令并行优化方法，树高平衡是其中比较重要的一个。作为一种局部方法，它可以重新组织 表达式树 ，以揭示更多指令层次的 并行性 。

引言

并行的重要性想必对于任何学习过计算机组成原理的人都无需多言，而并行编译的动机就是通过特定的指令重排操作，在不影响程序正确性的情况下让指令流能够包含多个 独立 的操作，使得其后续转化为的机器码可以在处理器的一个时钟周期中同时执行。在中端 CodeGen 部分，我们需要完成的是独立化的工作，至于具体的并行执行属于后端的编码工作。

这里的 树 指的是表达式树，此树的结构指示了中间代码生成的顺序和约束。我们在学习语法制导翻译时，了解过基于左递归和右递归方法的语法树绘制方法，对于某些程序场景，前端翻译直接得到的表达式树是不平衡的。例如，考虑 a + b + c + d + e 这段代码，左递归和右递归得到的语法树和对应的中间代码分别如下图所示：

树高平衡完成的工作就跟这个例子类似 —— 利用运算的交换律和结合律，通过重排计算指令的语法树为一个更加接近平衡树的结构，揭示表达式求值中可能的并行性（指令级并行）。

算法设计

树高平衡算法可以概括为 分析 → 变换 的两阶段操作：

  • 分析：识别程序块中的候选表达式树
    • 注意这是一个在 CodeGen 环节完成的局部优化，操作的对象是基本块及其包含的表达式语句，而不是前端的语法制导翻译，前面的例子只是为了解释指令级并行的潜在可能性
    • 候选树的条件
      • 所有运算符必须相同
        • 这意味着，候选树的本质实际上是一个 n 元运算符
      • 运算符必须可交换和可结合
        • 运用了全局优化技术 GVM 后，基本上所有可操作的加法 / 乘法运算语句都会被提到基本块前部，便利了这个优化的执行
        • 对于规模比较大的代码，候选树的规模也会很大，这将提供更多的重排机会，因此，算法应当试图构造 最大规模 的候选树
      • 标记候选树内部节点的每个名字不能重复
        • SSA 机制解决了一部分问题，但我们仍需保证基本块内部被使用多次的那些值要得到保留
      • 不可以改动基本块内部的可观察值
        • 可观察值的含义是在基本块外部可读取的那些值
  • 变换
    • 对于所有的候选树，找到所有运算对象，为其分配等级（等级的值是指令对应运算符的优先级），并将所有运算对象输入到一个优先队列，按等级递增的次序排列
    • 根据优先队列重建近似平衡二叉树的树
    • 将树的结构转化为指令序列，完成指令重排

具体实现

在查找树阶段，我们需要先对基本块中的每个变量  维护一个集合  ，记录基本块中所有使用了该变量作为 Lhs 或者 Rhs 的指令的索引值。如果该变量是一个可观察值，还需要往该集合中加入一个大于基本块中指令总数的整数 （ 的具体取值可以是任意的）。可见，如果  是一个局部临时变量，那么  的规模是 1 . 显然，一个非局部临时变量可能被其他指令二次利用，使用了它的那条指令 不可能是独立的 ，不可以作为候选树的叶子节点。

叶子节点的标准是独立，不存在（对本运算符而言的）指令依赖。

算法的第一阶段是找到基本块内 所有可能 的候选树，不论规模大小。具体而言，是找到所有可能的根节点，然后将其输入到一个优先队列中，该队列按照根节点运算符的优先级排序。

假设目标基本块有 n 条指令，每条指令都是形如 t[i] ← lhs[i] Op[i] rhs[i] 的三地址指令，识别根节点的伪代码如下所示：

可见，一个指令的运算对象能作为根节点的标准是：

  • 使用了非局部临时变量
  • 使用了局部临时变量  ，但赋值该临时变量的运算符  跟当前的运算符  不同，这说明运算符  是一个根节点，因为它既不可能在运算符  所属的候选树中再有父节点（  集合的规模 = 1 的含义），也不可能是运算符  所属的候选树的一部分
    • 注意这里是在分析指令  时判断的，判定为根节点的是指令 

从这里可以发现，优先队列的作用是维护指令之间变量依赖关系隐含的 操作符优先规则 ，保证优先级更高的计算先执行，针对它的指令重排不会影响到后面优先级更低且使用了有关变量的指令的计算。

第二阶段的内容是重构程序块为平衡树。此时以候选树根节点队列作为输入，对每个根节点都建立一个大体上平衡的树。伪代码如下所示：

在设定操作数优先级的步骤中，算法的策略是：

  • 常数等级为 0 ，迫使它们移动到优先队列的最前端
    • 通过这一步，实际上可以在树高平衡优化的步骤中完成一部分常数折叠的工作
  • 叶子节点等级为 1
  • 内部节点的等级为其所在子树所有节点的等级之和

这个指派策略可以生成近似于平衡二叉树的树状结构。
