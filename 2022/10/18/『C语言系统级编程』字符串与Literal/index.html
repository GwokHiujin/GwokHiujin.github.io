而对于情况 2，则稍有不同。此时 “hello” 在 静态存储区 分配了相应的空间。作为一个用双引号修饰的字符串（被称为 String Literal ），它之后有一个自动补充的隐藏字符 \0，并且具有静态存储周期（static storage duration）。

C 语言有 4 种 对象存储周期 ：static、automatic、allocated 和 thread。它们标识的是变量的存储持续性。

  1. 自动 存储持续性：在函数定义中声明的变量，包括函数参数的存储持续性为自动的，它们在程序开始执行其所属的函数或代码块时被创建执行完函数或代码块时其内存被释放，C++ 中有两种持续性为自动的变量。
    1. 如果对象的标识符声明时没有链接，也没有标识符 static，那么对象的存储周期就是自动的。
  2. 静态 存储持续性：在函数体外定义的变量和使用关键字 static 定义的变量的存储持续性都是静态，它们在整个程序运行过程中都存在。需要注意的是，它的存储值只会在程序启动之前初始化一次。
  3. 线程 存储持续性：被关键字 thread_local 声明的变量的生命周期与其所属的线程一样长。
  4. 动态 存储持续性：用 new / malloc 申请的内存一直存在直到 delete / free 将其释放或进程结束，这种内存的存储持续性为动态，又称自由存储或堆。

一个程序的存储类型和它的作用域，生存期限及连接类型具有某种微妙的关系，但一个具有作用域和连接类型的标识符不一定就具有存储类型。

这四种存储类型可分为两种生存期限：永久（程序执行期间均存在）和临时（暂时保存在堆栈和寄存器中）。

String Literal 和 String

String Literal 的官方定义是被双引号括起来的字符串里的所有合法字符，顾名可以思义。合法字符当然包括空字符 \0，也就是说 “hello\0world” 也是一个 String Literal。

但它不能够是一个 String。String 的官方定义是一个连续的字符序列，以第一个空字符 \0 结束并包含第一个空字符。也就是说 String 有且只有末尾那个隐藏的字符是 \0，如果出现在中间会被当作截断。

注意 String Literal 和 String 的末尾空字符都是隐藏的。

A character string literal is a sequence of zero or more multibyte characters enclosed in double-quotes.

A string is a contiguous sequence of characters terminated by and including the first null character.

不失一般性，可以将 String 和 String Literal 都统称为字符串。我们一般所说的字符串指的都是 String。

  • sizeof(String Literal) vs strlen(String Literal)

讨论 String Literal 的长度时，值得注意的问题是中间的空字符是否要被考虑。sizeof 函数计量的是所占内存的大小，因此 sizeof(String Literal) 会计入中间的空字符，包括末尾隐藏的那个空字符；而观察 strlen 的源码就可以知道它对字符串长度的判断逻辑是从开始到第一个空字符为止，且不包括那个空字符，因此需要注意它俩输出结果的区别。

定位内存

用 双引号引导 的字符串可以用来 定位 这块内存。以 “hello” 为例，其含义包括：

  1. 在静态存储区找到合适的空间，存放 ‘h’, ‘e’, ‘l’, ‘l’, ‘o’, ‘\0’
  2. 用 “hello” 可以定位到这块在静态存储区中的内存

因此 “hello” 的内存六元组可以表示为：

如果要对 “hello” 取值，返回值就应该是 <0x00404039, char*>。这里可以引出一个好玩的题目： sizeof(“hello”) vs sizeof(“hello”+1-1) 。根据六元组定义，sizeof(“hello”) 的结果是 6，因为可以定位到 “hello” 对应的内存，获得这块内存的大小；而 “hello” + 1 - 1 由于涉及到计算，会对 “hello” 取值，这块内存的表示值类型是 char*，进行了 +1 和 -1 操作后不会改变表达式返回值的类型，因此返回的值是 4，等于 sizeof(char*)。

所有指针类型的 size 都是 4。

能定位内存，说明双引号引导的字符串是一个左值（lvalue），根据前面学过的左值相关知识，说明我们可以将其看成一个 exp 进行处理。此时又可以引出一个问题：既然双引号引导的字符串是一个类型为 char* 的左值表达式，我们能否对其做出形如 char c = “hello”[0] 或者 “hello[0] = 'e'” 这样的操作呢？

第一种是合法的。我们可以通过 * 定位取出 “hello”[0] 的值 <‘h’, char> 并赋给 c，具体步骤跟之前的定位数组元素是一样的。

但第二种会出现问题。我们知道 “hello” 是存储在静态存储区中的，而 C 语言规定修改静态存储是一种未定义行为，其中的值是 not modifiable 的。因此这种赋值操作形式上合法，但实际上是未定义行为。

静态存储的静态特性（static）还会导致一个问题，即 char* p = “hello” 和 char*q = “hello” 中的 p 和 q 是完全相同、完全等价的，修改其中一个会联动着影响到另一个。当然，实践中并不会有人真的直接使用双引号引导的字符串去做什么事情。

Compound Literal

复合字面量 Compound Literal 的含义是用 (type-name){Initializer-list} 的格式声明定义一个匿名（unnamed）的对象，该对象的变量类型是 type-name，由 Initializer-list 进行初始化。这是一种合法的后缀表达式，Literal 在这里强调按字面理解的 type + initializer 的组合形式。

比如 (int){1} 就声明定义了一个变量类型为 int ，初始化值为 1 的对象。如果将其定义在函数外，它就拥有静态存储周期（static storage duration），否则拥有的是自动存储周期（automatic storage duration）。在拥有自动存储周期时 (int){1} 是合法的 modifiable lvalue，意味着你可以对它进行诸如 (int){1} = 5 的操作；而当它是 static storage duration 的时候就不行了，不是合法的全局变量始化。

匿名变量最大的问题在于：没有 identifier 的话怎么访问这块内存呢？通过上面的 String Literal 应该也能推断出，Compound Literal 本身就是一个能定位内存的左值表达式。

在同一个作用域内，同样的 Compound Literal 只有一个。

对于数组类型的 type-name 感觉没什么好说的，正常观察内存六元组即可。

Literal 的含义

说到底，Literal 到底是什么东西？这个单词的基本释义是这样的：

  1. being the most basic meaning of a word or phrase,
    rather than an extended or poetic meaning
  2. that follows the original words exactly
  3. lacking imagination

荣老师说他不喜欢“字面量”这个翻译，但我觉得还挺好的……就是将一个“字面意义”上的东西，或者按直接的字面意义去理解的东西作为可用的左值表达式。C 语言中只有两个 Literal 修饰的概念，就是上面讲了的 String Literal 和 Compound Literal；而在 C++ 中，所有被称为 “constant” （常量）的 tokens 都被称作 Literal。

以 String Literal 为例，一般谈论字符串 String 的时候认为有且只有最后一个字符是 \0 ，而为字符串中间的空字符赋予了“截断字符串”的意义，但空字符本身也是一个有效的字符，因此只按字面意义去理解的话，一个中间还包含多个空字符的字符串也应该从字面意义的角度被视为一个完整的整体，因此就有了 String Literal 的概念去包容它。

下面特别说明一下 常量 （constant）的概念。C 语言包括 4 种常量：整数常量、浮点数常量、字符常量和枚举常量。注意枚举常量和整数型字符常量的 type 都是 int，这就导致：

这个怪现象在 C 和 C++ 环境下表现不一致，在 C++ 中 sizeof(‘a’) 就是 1。针对这一特点正在讨论是否要发起修改提案。

常量与 Literal 的区别在于：

常量和 const 修饰的对象也不能一概而论，它们属于完全不同的概念体系。const 修饰的对象如果没有被 volatile 同时修饰，则：

  1. 可以把这个对象放到只读区域（read-only region of storage）
  2. 如果没有对这个对象有取地址的操作，可以不用真的为这个对象分配内存

如果被 volatile 修饰了的话，该对象的 identifier 对应的内存虽然由 const 修饰，但仍然可能会被修改。

constant 在 C++ 中就被化为对应的 literal 了。其中：

  • String Literal in C vs string-literal in C++
    • C++ 中双引号引导字符串的返回值类型是 const char*，是为了对应 Literal 在 C++ 中具有常量的含义
    • C++ 中形如 “hello”[0] = ‘a’ 的操作不是未定义行为，而是会直接编译报错
  • Compound Literal in C vs …?
    • C++ 中没有 Compound Literal 的概念（但那个形式还是一个合法的后缀表达式），因为无法赋予其一个常量的含义

对象的存储周期

任何对象都有生命周期（lifetime），决定生命周期的就是对象的存储周期（Storage Duration）。在对象的生命周期内：

  • 系统确保对象的内存有效
  • 对象的地址是 constant address ，在对象声明周期内，地址不变
  • 对象保有的最后赋值（last-stored value）不变

超出对象生命周期之外对对象的访问是未定义行为。

C 语言定义了如下的 Storage-class specifier，可以根据它们声明对象存储周期：

  • typedef
    • 没有定义新的对象类型，只是提供更好的书写便利性
  • extern
    • 一般用在变量名前或函数名前，作用是用来说明“此变量 / 函数是在别处定义的，要在此处引用”
    • 如果在所有函数之外定义的变量没有指定其存储类别，那么它就是一个外部变量，它的作用域是从它的定义点到本文件的末尾（ 在单个源文件中的确是这样，如果有多个源文件，全局变量的作用范围不是从变量定义处到该文件结尾，而是在其他文件中也有效） 。但是如果要在定义点之前或者其他文件中使用它，那么就需要使用关键字 extern 对其进行声明（注意不是定义，编译器并不为其分配内存）
      • 如果在声明的时候给变量赋值，那么就和去掉 extern 直接定义变量赋值是等价的
      • 声明可以多次，定义只能一次
    • 外部变量保存在静态存储区内，在程序运行期间分配固定的存储单元，其生存期是整个程序的运行期，没有显式初始化的外部变量由编译程序自动初始化为 0（extern 说明符的用途是 扩展 全局变量的作用域，可以将全局变量的作用域扩展到其他文件， 但不能限制全局变量的作用域 ）
    • https://www.cnblogs.com/lanhaicode/p/10633125.html
  • static
  • thread_local
  • auto
    • 由于 auto（自动变量）极为常用，所以 C 语言把它设计成缺省的存储类型，即 auto 可以省略不写
  • register
    • 告诉编译器越快越好，但编译器可以不处理
    • register 修饰的对象不能取地址

标识符的作用域

标识符（Identifier）是 C 语言中最基础的一种 Symbol，它可以用来识别：

an object; a function; a tag or a member of a structure, union, or enumeration;

a typedef name; a label name; a macro name; or a macro parameter.

对于标识符所指定的每个不同实体，该标识符仅在程序文本中被称为其作用域（Scope）的区域内可见（即可以使用）。

C 语言中共有 4 种 Scope：

  • function

    • 标签名是唯一拥有函数作用域的标识符。比如下图中的标签名 skip：

  • file

    • 如果声明标识符的声明器或类型说明符出现在 任何块或参数列表之外 ，则标识符具有文件作用域，该作用域在当前翻译单元结束时终止。
      • 翻译单元指 C 程序预处理过后的输出。一个 C 程序不需要同时翻译所有的代码，程序的文本保存在称为源文件的单元中，通过预处理指令 #include 包含的所有头文件和源文件一起被称为预处理翻译单元。预处理后的预处理翻译单元称为翻译单元。
      • 这说明文件范围即翻译单元的范围，属于一个内部链接（internal linkage，下面会讲），内部链接中的函数和变量对于链接器（linker）是不可见的，也即对于其他目标文件不可见。

  • block

    • 如果声明标识符的声明符或类型说明符出现在 块中或函数定义中的参数声明列表中 ，则标识符具有块作用域，该作用域终止于关联块的末尾。

  • function prototype

    • 当标识符出现在函数原型中的 参数声明列表 中（不是函数定义的一部分）时，它具有函数原型作用域，其生命周期在函数声明符的末尾终止。

    • 注意区分函数原型和函数定义！所谓声明（Declaration），就是告诉编译器我要使用这个函数，你现在没有找到它的定义不要紧，请不要报错，稍后会把定义补上。

      函数声明的格式非常简单，相当于去掉函数定义中的函数体，并在最后加上分号，如下所示：

      dataType functionName( dataType1 param1, dataType2 param2 ... );

      也可以不写形参，只写数据类型：

      dataType functionName( dataType1, dataType2 ... );

      函数声明给出了函数名、返回值类型、参数列表（重点是参数类型）等与该函数有关的信息，称为函数原型（Function Prototype）。函数原型的作用是告诉编译器与该函数有关的信息，让编译器知道函数的存在，以及存在的形式，即使函数暂时没有定义，编译器也知道如何使用它。

Linkage

如果相同的标识符出现在不同的地方，C 语言需要依靠 Linkage 机制分辨它们是否表示同一个实体。C 语言有 3 种 Linkage：

  • external
    • 对于使用了存储类说明符 extern 进行声明的标识符，在该标识符的先前声明可见的范围内，如果先前声明指定了内部或外部链接，则在后一声明中标识符的链接与先前声明中指定的链接相同。如果没有可见的预先声明，或者预先声明没有指定链接，则标识符具有外部链接。
  • internal
    • 如果对象或函数的文件范围标识符的声明包含存储类说明符 static ，则标识符具有内部链接。
    • 只有在文件作用域内，函数声明才可以包含静态存储类说明符
  • none
    • 以下的标识符不含 linkage：
      • 声明为对象或函数以外的任何东西的标识符；
      • 声明为函数形参的标识符；
      • 没有使用存储类说明符 extern 来声明对象的块范围（block scope）标识符
    • no linkage 的标识符都指向不同的对象
